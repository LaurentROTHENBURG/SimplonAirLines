Description :

Il s'agit du second brief. le sujet est le suivant : c'est la crise sanitaire et votre entreprise vous demande de réaliser un prototype de jeu de plateau en mode console avant de développer sa version en réseau. L'objectif du brief est de programmer en JAVA

Conception :

Le projet s'est réalisé en mode pair programming (Lien vers Wiki) .Avant le développement,il a été nécessaire de définir la structure du développement. Différentes étapes logiques devaient réaliser le programme. Lorsque la structure du programme a été posée, nous sommes passés au dévelopement en binome, en changeant régulièrement de rôle,driver puis observeur Initialisation des couleurs, Premier affichage du tableau, Fonctions de test de la grille, Test de la ligne,

Codage :

Le code a été réalisé en JAVA. Le code met en place un tableau à deux dimensions, des boucles pour éditer le tableau ainsi que des conditions If. Le code permet également de controler la saisie de l'utilisateur. La complexité du code réside dans la procédure d'alignement des 4 jetons.



Exemple de code permettant d'afficher le tableau:

for (int i=0; i< game.length; i++){
System.out.print("| ");
for(int j=0; j< game[i].length; j++){
if(game[i][j] == null){
System.out.print(" " + " | ");
}
else {
//Affichage d'une croix ou d'un rond
switch (game[i][j]) {
case "1":
System.out.print(ANSI_RED + "X" + ANSI_RESET + " | "); break;
case "2" :
System.out.print(ANSI_YELLOW + "O" + ANSI_RESET + " | "); break;
}
//System.out.print(game[i][j] + " | ");
}
}
System.out.print("\n");
}

Résultat :

Le résultat est satisfaisant ! Le programme propose aux joueurs de choisir une colonne contenue dans le tableau et vérifie l'alignement des 4 jetons honrizontalement, verticalement, et en diagonale
